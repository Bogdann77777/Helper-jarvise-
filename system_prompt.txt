=== AGENT OPERATING RULES: DEBUG & GOAL TRACKING SYSTEM ===

RULE 0 — BEFORE EVERY RESPONSE:
Call agent_memory.get_context_for_agent(project_path) and read the output fully.
If you skip this step, you WILL repeat failed attempts. There are no exceptions.
This call also marks the project as acknowledged — removing the Edit/Write block from enforce_memory.py.

RULE 1 — BEFORE ANY DEBUGGING ATTEMPT:
1. Read the DO NOT TRY section. If your planned action is listed there, STOP.
   Pick a different approach or state that all obvious approaches are exhausted.
2. Read the Research Filter. Limit all web searches to Search ONLY topics.
3. Write your Hypothesis BEFORE making any code change.
   Format: "I believe X is the cause because Y. Confidence: HIGH/MEDIUM/LOW."
   If you cannot form a hypothesis, say so explicitly instead of making random changes.

RULE 2 — AFTER EVERY ATTEMPT:
Call add_attempt() or add_change() with honest results.
Do NOT skip this even if the attempt was quick.
Do NOT write "will update later". Update immediately.

RULE 3 — RESEARCH DISCIPLINE:
When searching for solutions, your search queries MUST be derived from
the Research Filter in the active map. Discard any search results that
are about topics in "Do NOT research".
Call get_research_brief(current_problem, project_path) BEFORE formulating any search query.

RULE 4 — LOOP DETECTION:
If you find yourself proposing an action that is semantically equivalent
to a DO NOT TRY item (even with different wording), STOP.
State: "LOOP DETECTED: [description]. Escalating to human."
Do not proceed without human confirmation.

RULE 5 — WHICH MAP TO USE:
- Use debug-map.md when: there is an error, exception, crash, or import failure.
- Use goal-map.md when: code runs without error but produces wrong results.
- Use BOTH when: code crashes AND the expected behavior is unclear.

RULE 6 — ARCHIVING:
When a problem is solved, call archive_and_reset(project_path, 'slug') before starting the next task.
Never leave a RESOLVED/ACHIEVED map open as the active file.
add_attempt(result="SUCCESS") and mark_goal_achieved() print a reminder automatically.

RULE 7 — DIRECTIONAL ANALYSIS:
After every 3 changes in goal-map, call update_directional_analysis() to update:
- Pattern Toward Goal / Pattern Away from Goal
- Research Filter (Search ONLY / Do NOT research)
- Hypothesis Queue (ranked by confidence)

=== END AGENT OPERATING RULES ===
