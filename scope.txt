# Техническое задание: Голосовой интерфейс для Claude CLI через локальную сеть

## Общая архитектура

**Принцип работы**: Веб-интерфейс на любом устройстве (телефон/ноутбук) → Wi-Fi → FastAPI сервер на мощном ПК → обработка голоса и команд на GPU.

---

## Компоненты системы

### 1. Серверная часть (на ПК с GPU)

**Железо**:
- ПК с 2x RTX 5060 Ti 16GB
- Операционная система: Windows
- PowerShell (желательно версия 7+ для лимита 32767 символов)

**Программная часть**:

#### 1.1 FastAPI сервер
- **Язык**: Python 3.10+
- **Порт**: 8000 (или любой свободный)
- **Host**: `0.0.0.0` (доступен из локальной сети)
- **Эндпоинты**:
  - `GET /` - веб-интерфейс (HTML страница)
  - `POST /api/voice` - прием аудио файла
  - `GET /api/status` - статус обработки (для отображения прогресса)
  - `WebSocket /ws` - для real-time обновлений статуса

#### 1.2 STT (Speech-to-Text)
- **Модель**: Whisper `large-v3` или `medium` (в зависимости от скорости)
- **Библиотека**: `faster-whisper` (CTranslate2 оптимизация)
- **GPU**: Использует одну из RTX 5060 Ti
- **Формат входа**: WebM/WAV аудио из браузера
- **Язык**: auto-detect (но в основном русский/украинский/английский)

#### 1.3 TTS (Text-to-Speech)
- **Модель**: XTTS v2
- **GPU**: Использует вторую RTX 5060 Ti (или та же, если Whisper быстро освобождает)
- **Формат выхода**: WAV/MP3 для браузера
- **Голос**: Настраиваемый (можно клонировать свой голос)

#### 1.4 CLI Executor
- **CLI**: Claude CLI от Anthropic
- **Запуск**: через PowerShell
- **Команда**: `claude` (или полный путь к исполняемому файлу)

**Ограничение PowerShell**:
- PowerShell 5.x: максимум ~8191 символ в одной команде
- PowerShell 7+: максимум ~32767 символов в одной команде
- **Решение**: Разбивка длинного текста на чанки

**Логика разбивки на чанки**:
```
ЕСЛИ длина_текста > ЛИМИТ_СИМВОЛОВ:
    1. Разбить текст на части по ЛИМИТ_СИМВОЛОВ - 500 символов (запас)
    2. Разбивать по границам предложений (не рвать слова)
    3. Отправлять чанки последовательно в Claude CLI
    4. Собирать все ответы в один итоговый ответ
ИНАЧЕ:
    Отправить весь текст одной командой
```

**Определение лимита**:
```powershell
# Проверить версию PowerShell
if ($PSVersionTable.PSVersion.Major -ge 7) {
    ЛИМИТ = 30000  # с запасом
} else {
    ЛИМИТ = 7500   # с запасом
}
```

### 2. Веб-интерфейс (клиентская часть)

**Технологии**:
- HTML5
- JavaScript (нативный, без фреймворков для простоты)
- WebRTC для записи микрофона
- WebSocket для real-time статуса

**Элементы интерфейса**:

1. **Кнопка записи** (большая, центральная)
   - Состояния: "Нажми и говори" → "Слушаю..." → "Обрабатываю..."

2. **Индикатор активности микрофона**
   - Визуализация уровня звука (чтобы видеть, что идет запись)

3. **Таймер тишины**
   - Показывает обратный отсчет от 5 секунд тишины до отправки

4. **Панель статуса/прогресса**
   - Показывает текущий этап:
     - "Запись голоса..."
     - "Распознавание речи... (3 сек)"
     - "Ваш текст: [распознанный текст]"
     - "Отправка в Claude CLI..."
     - "Чанк 1/3 отправлен..."
     - "Получен ответ от Claude..."
     - "Генерация голоса..."
     - "Готово! Слушайте ответ."

5. **Окно транскрипта**
   - Показывает распознанный текст ДО отправки
   - Накапливает весь текст до паузы 5 секунд
   - Можно редактировать перед отправкой (опция)

6. **Окно ответа Claude**
   - Текстовый ответ от CLI
   - Streaming вывод (если возможно показывать по мере получения)

7. **Аудио плеер**
   - Автоматически воспроизводит голосовой ответ
   - Можно переслушать

**Доступ из локальной сети**:
- IP-адрес ПК в локальной сети (например `192.168.1.100`)
- URL: `http://192.168.1.100:8000`
- Работает на любом устройстве: телефон, планшет, ноутбук

## Логика работы (пошаговый сценарий)

### Шаг 1: Запись голоса

```
ПОЛЬЗОВАТЕЛЬ нажимает кнопку "Начать запись"
  ↓
БРАУЗЕР начинает запись с микрофона (WebRTC)
  ↓
ИНДИКАТОР показывает уровень звука
  ↓
ИДЕТ НАКОПЛЕНИЕ аудио в буфер
  ↓
ДЕТЕКТОР ТИШИНЫ мониторит уровень звука:
  - Если звук есть → сбросить таймер тишины
  - Если тишина > 5 секунд → АВТОМАТИЧЕСКИ завершить запись
  
АЛЬТЕРНАТИВА: 
  - Пользователь говорит стоп-слово "СТОП" или "ГОТОВО"
  - Пользователь нажимает кнопку "Завершить"
```

### Шаг 2: Отправка на сервер

```
БРАУЗЕР отправляет аудио файл через POST /api/voice
  ↓
СТАТУС в интерфейсе: "Отправка аудио на сервер..."
  ↓
СЕРВЕР принимает аудио
```

### Шаг 3: Распознавание речи (STT)

```
СЕРВЕР → СТАТУС: "Распознавание речи..."
  ↓
WHISPER (на GPU) обрабатывает аудио
  ↓
ПОЛУЧЕН ТЕКСТ (например: "Привет, Claude! Напиши мне функцию для сортировки массива...")
  ↓
СЕРВЕР → СТАТУС: "Распознано: [показать первые 100 символов]..."
  ↓
WebSocket отправляет полный текст в браузер для отображения
```

### Шаг 4: Подготовка команды для Claude CLI

```
ПРОВЕРКА длины текста:
  
ЕСЛИ текст.length > ЛИМИТ_POWERSHELL:
    СТАТУС: "Текст длинный, разбиваю на части..."
    
    Разбить на чанки:
      чанк_1 = первые 7500 символов (по границе предложения)
      чанк_2 = следующие 7500 символов
      ...
      чанк_N = остаток
      
    СТАТУС: "Разбито на {N} частей"
    
ИНАЧЕ:
    чанков = 1
```

### Шаг 5: Выполнение в Claude CLI

```
ДЛЯ КАЖДОГО чанка:
  
  СТАТУС: "Отправка части {i}/{N} в Claude CLI..."
  
  Выполнить PowerShell команду:
    pwsh -Command "claude '{экранированный_текст_чанка}'"
  
  СТАТУС: "Обработка части {i}/{N}... (Claude думает)"
  
  Дождаться ответа от CLI
  
  СТАТУС: "Получен ответ части {i}/{N}"
  
  Сохранить ответ в буфер

КОНЕЦ ЦИКЛА

Объединить все ответы в один итоговый текст
```

**Важно**: 
- Экранировать спецсимволы для PowerShell (кавычки, бэкслэши и т.д.)
- Обрабатывать ошибки CLI (таймауты, сбои)
- Логировать все команды и ответы

### Шаг 6: Генерация голосового ответа (TTS)

```
СТАТУС: "Генерирую голосовой ответ..."
  ↓
XTTS v2 (на GPU) генерирует аудио из текста ответа
  ↓
СОХРАНИТЬ аудио файл (WAV/MP3)
  ↓
СТАТУС: "Голос готов!"
```

### Шаг 7: Отправка результата клиенту

```
СЕРВЕР отправляет JSON ответ:
{
  "transcript": "распознанный текст пользователя",
  "response_text": "текстовый ответ от Claude",
  "audio_url": "/audio/response_12345.mp3",
  "chunks_sent": 3,
  "processing_time": "12.4 секунд"
}
  ↓
БРАУЗЕР отображает:
  - Текстовый ответ в окне
  - Автоматически воспроизводит аудио
  ↓
СТАТУС: "Готово! Можете задать следующий вопрос."
```

## Детекция конца речи (VAD - Voice Activity Detection)

**Метод**: Порог тишины

```javascript
// В браузере
let silenceStart = null;
const SILENCE_THRESHOLD = 5000; // 5 секунд
const AUDIO_THRESHOLD = 0.01;   // порог громкости

function analyzeAudioLevel(audioData) {
  const level = calculateRMS(audioData); // среднеквадратичное значение
  
  if (level < AUDIO_THRESHOLD) {
    // Тишина
    if (!silenceStart) {
      silenceStart = Date.now();
    } else if (Date.now() - silenceStart > SILENCE_THRESHOLD) {
      // Прошло 5 секунд тишины → ЗАВЕРШИТЬ ЗАПИСЬ
      stopRecording();
      sendAudioToServer();
    }
  } else {
    // Есть звук → сбросить таймер
    silenceStart = null;
  }
}
```

**Визуализация**: 
- Показывать обратный отсчет: "Тишина: 5... 4... 3... 2... 1... ОТПРАВКА!"

## Отображение прогресса в реальном времени

**Технология**: WebSocket соединение

```
СЕРВЕР → КЛИЕНТ через WebSocket:

{
  "type": "status_update",
  "stage": "stt",
  "message": "Распознавание речи...",
  "progress": 30  // процент (опционально)
}

{
  "type": "status_update",
  "stage": "cli_execution",
  "message": "Отправка чанка 2/3 в Claude CLI...",
  "progress": 60
}

{
  "type": "cli_output",
  "data": "Claude начал отвечать...\n" // streaming вывод
}

{
  "type": "status_update",
  "stage": "tts",
  "message": "Генерация голоса...",
  "progress": 90
}
```

**Этапы (stages)**:
1. `recording` - Запись голоса
2. `uploading` - Загрузка на сервер
3. `stt` - Распознавание речи
4. `cli_prep` - Подготовка команды
5. `cli_execution` - Выполнение в Claude CLI
6. `tts` - Генерация голоса
7. `complete` - Готово

## Обработка ошибок

**Возможные ошибки**:

1. **Микрофон недоступен** → Показать инструкцию по разрешению доступа
2. **Нет подключения к серверу** → "ПК недоступен, проверьте подключение"
3. **Whisper не смог распознать** → "Не удалось распознать речь, попробуйте еще раз"
4. **Claude CLI вернул ошибку** → Показать текст ошибки
5. **Таймаут Claude CLI** → "Claude не отвечает, попробуйте позже"
6. **XTTS не смог сгенерировать голос** → Показать текстовый ответ без аудио

**Логирование**:
- Все ошибки сохранять в лог-файл на сервере
- Критичные ошибки отправлять в WebSocket клиенту

## Безопасность

1. **Локальная сеть только** - сервер не доступен из интернета
2. **CORS** - только с локальных IP разрешены запросы
3. **Нет аутентификации** (внутри домашней сети не критично, но можно добавить базовый пароль)
